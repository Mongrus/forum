# SerenkoDev Forum

Полнофункциональное форумное приложение, разработанное на современном fullstack-стеке Laravel + Inertia + Vue. 
Проект демонстрирует реализацию реального продакшн-решения с разграничением прав доступа, системой модерации,
real-time коммуникацией и продуманной архитектурой.

Продакшн-версия:  
https://forum.serenkodev.ru/

---

## Технологический стек

### Backend
- Laravel
- Laravel Breeze (регистрация и авторизация)
- Redis
- WebSockets
- Очереди (Queues)
- Частичная реализация Service / Repository слоёв с интерфейсами

### Frontend
- Inertia.js
- Vue 3
- Vite
- Pinia
- Sass (БЭМ-методология)
- Адаптивная верстка (rem + mixins)

### Инфраструктура
- VPS-сервер
- Ubuntu (production)
- Тестирование на macOS и Linux
- Выкупленный домен и домен третьего уровня
- Ручная настройка окружения и деплоя

---

## Архитектура форума

Иерархическая структура данных:
Раздел
└── Тема
└── Пост
└── Комментарии


Новые статьи проходят модерацию перед публикацией.

---

## Аутентификация и авторизация

- Регистрация и вход реализованы через Laravel Breeze
- Разделение маршрутов для:
  - Гостей
  - Авторизованных пользователей
- Ролевое разграничение доступа:
  - Администратор
  - Модератор
  - Пользователь
- Административные маршруты защищены:
  - Через middleware-группы
  - Через условный рендеринг во фронтенде

---

## Роли и права доступа

Разделение ролей влияет на:

- Доступ к админ-панели
- Создание новых разделов форума
- Модерацию статей
- Обработку обращений в техническую поддержку
- Редактирование и удаление постов пользователей

Пользователи не имеют доступа к созданию разделов и административным маршрутам.

---

## Система бана

- Реализована система блокировки пользователей
- Ограничение доступа к защищенным маршрутам
- Заблокированному пользователю доступно только:
  - Отправка обращения в техническую поддержку
- Обращения направляются в специальную категорию на модерацию

---

## Real-time функциональность

Реализовано с использованием WebSockets и Redis:

- Обновление чатов в реальном времени
- Отображение события «пользователь печатает»
- Синхронизация онлайн-статусов
- Хранение статусов в Redis
- Отображение статуса пользователя на всех страницах форума

- Дополнительные сущности для ограничения количества действий
